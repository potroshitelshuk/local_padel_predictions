## Описание

**Исследовательский вопрос** Какие паттерны игровой активности (рейтинги, rolling-статистики матчей, совместный опыт пары, head-to-head, форма игроков, отдых, контекст, matchup) предсказывают **локальные исходы** матча - победителя первого сета, наличие третьего сета, наличие тай-брейка и т.д.? Насколько эти исходы взаимосвязаны и насколько они помогают улучшить предсказательную способность ML моделей по построению прогноза победитля матча?

**Этапы:**

1. **Feature engineering паттернов активности** (`02_feature_engineering/`). 8 тематических блоков фич, состоящие из отдельного ipynb ноутбука с сохранением в CSV
2. **Pre-match бейзлайн** (`03_baseline/`). Для каждого блокапризнаков смотрим, какую долю предсказательной силы даёт именно он. Это бейзлайн, который мы потом используем для интерпретации Level-1
3. **Level-1 - предсказание локальных исходов** (`04_local_outcomes/`). Для 5 локальных исходов строим отдельные модели на pre-match признаках. Предсказания генерируются через expanding window по турнирам 2024-2026. Для каждого таргета анализируется: качество, калибровка, feature importance (какие паттерны активности объясняют этот исход), взаимная корреляция между исходами
4. **Level-2 - эксперимент** (`05_enhanced_model/`). Проверяем, несут ли L1-предсказания дополнительную информацию сверх pre-match признаков для предсказания **итогового** победителя матча

**Временная схема:**

```
──── 2023 ──── 2024 ──── 2025 ──── 2026 (янв-мар) ────
                │
                ▼ expanding window (переобучение перед каждым турниром)
Level-1 pred:   [ 2024 ─────── 2025 ─────── 2026_Q1 ]

Level-2 train:  [ 2024 ───── 2025 без N турниров в конце 2025 ]
Level-2 test:                    [ хвост 2025 + 2026_Q1 ]
```

Перед каждым турниром 2024-2026 Level-1 модели переобучаются на всей истории до даты этого турнира. Level-2 обучается на 2024 + 2025 без N турниров в конце 2025 и тестируется на хвосте 2025 + 2026_Q1

**Данные:** только мужские матчи Premier Padel.  
**Источник данных:** [Padel API](https://docs.padelapi.org/).

## План работ

### Чекпойнт 1: Обзор источников

- Сбор и анализ релевантных источников (7-10 источников)
- Выделение метрик, базовых и SOTA-моделей
- Анализ инженерных решений и артефактов
- Согласование с научным руководителем

### Чекпойнт 2: Данные и EDA *(готово)*

- Получение доступа к Padel API
- Сбор данных по матчам Premier Padel (мужские, 2023-2026)
- Exploratory Data Analysis

### Чекпойнт 3: Feature Engineering *(готово)*

Каждый блок реализован в отдельном ноутбуке, выходной CSV сохраняется в `data/features/<block>.csv` с ключом `match_id`. Блоки независимы и мёржатся по ключу на этапе моделирования

- Rolling-статистики (обобщённая функция, team-level + player-level)
- Elo-рейтинги (индивидуальный, парный, margin-weighted)
- Pair chemistry (совместный опыт пары, win rate пары)
- Head-to-head (пара-vs-пара и по игрокам)
- Форма и серии (win rate за N последних матчей, текущая серия, качество соперников, Elo momentum)
- Отдых и нагрузка (rest days, кол-во матчей за N дней, кол-во геймов за N дней)
- Контекст (home advantage, история в этом турнире, календарь)
- Matchup (комбинация рук, разницы в росте/возрасте внутри пары)

### Чекпойнт 4: Pre-match модель и вклад групп признаков *(готово)*

- **Dummy** и **Static-only** модель как первые бейзлайны
- Для каждого блока фич: модель `Static + блок` (оценка вклада каждой группы паттернов активности)
- **Full pre-match** модель: все блоки вместе, бейзлайн для Level-2 модели
- Итоговая таблица: вклад каждой группы и финальные метрики бейзлайна

### Чекпойнт 5: Предсказание локальных исходов (Level-1) *(почти готово, нужно больше экспериментов с таргетами)*

Таргеты покрывают разные грани матча: раннее доминирование, плотность счёта и общее доминирование

- `set1_winner_team1` (binary), `has_3rd_set` (binary), `any_tiebreak` (binary)
- `set1_margin_games` (regression), `games_diff_total` (regression)
- Генерация прогнозов с expanding window (переобучение перед каждым турниром 2024-2026), train swap-аугментация
- **Feature importance per target:** какие паттерны игровой активности объясняют каждый локальный исход
- **Матрица корреляций** между предсказаниями L1: насколько исходы ортогональны

### Чекпойнт 6: Прикладной эксперимент - Level-2 match outcome *(почти готово, нужно больше экспериментов с вариантами обучения)*

Проверка результата: несут ли L1-предсказания дополнительную информацию сверх исходных pre-match признаков

- Обучение L2 модели с L1-предсказаниями как дополнительными фичами
- Тестирование групп признаков: L1-only / Static + L1-only / binary-2 / regression-2 / binary-3 / all-5 vs Baseline (Full pre-match)
- Feature importance L2 и разбор матчей, где Enhanced изменил предсказание Baseline

### Чекпойнт 7: Финализация

- Сравнительный анализ всех подходов
- Интерпретация результатов
- Оформление работы
